deepseek给出的建议：

你遇到的错误是由于使用了过时的 Python 语法。具体来说，except KeyboardInterrupt, e: 这种写法在 Python 2.x 中是有效的，但在 Python 3.x 中已经被弃用了。在 Python 3.x 中，你应该使用 except KeyboardInterrupt as e: 这种语法。

解决方法：

  1. 检查 Python 版本：
    首先，确认你当前使用的 Python 版本。你可以通过以下命令查看：

    如果输出是 Python 3.x，那么你需要更新 yum 脚本以适应 Python 3 的语法。

  2. 更新 yum 脚本：
    如果你有权限编辑 /usr/bin/yum 文件，可以手动将 except KeyboardInterrupt, e: 修改为 except KeyboardInterrupt as e:。

  3. 安装 yum 的 Python 3 版本：
    如果你使用的是 CentOS 8 或更高版本，yum 已经被 dnf 取代，并且 dnf 是基于 Python 3 的。你可以尝试使用 dnf 来代替 yum：

  4. **使用 Python 2 运行 yum**：
    如果你仍然需要使用 yum，并且系统中同时安装了 Python 2 和 Python 3，你可以尝试使用 Python 2 来运行 yum：

  5. 更新系统：
    如果你使用的是较旧的 CentOS 版本（例如 CentOS 7），考虑升级到 CentOS 8 或更高版本，因为这些版本已经完全迁移到 Python 3。

总结：

  • 如果你使用的是 CentOS 8 或更高版本，建议使用 dnf 代替 yum。
  • 如果你使用的是 CentOS 7 或更早版本，并且需要继续使用 yum，可以尝试手动修改 /usr/bin/yum 文件中的语法错误，或者使用 Python 2 来运行 yum。
