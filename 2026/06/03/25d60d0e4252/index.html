Tabbit——重新定义浏览器的AI原生新物种

時光2026-06-032026-06-13

一、什么是 Tabbit？

Tabbit 是由美团旗下 GN06 团队（光年之外）推出的 AI 原生浏览器，于 2026 年 3 月 2 日开启公测，同年 6 月 9 日正式发布 1.0 版本。

与传统浏览器加装 AI 插件的思路不同，Tabbit 从底层重新设计了浏览器与 AI 的交互逻辑——它把浏览器本身变成了一个可以被 AI 理解和驱动的作业环境，而非仅仅在侧边栏加了一个聊天框。

核心数据：

  • 公测 100 天，每周迭代，累计更新超百项功能
  • Agent 自动化任务成功率从公测初期的 53.1% 提升至 1.0 版本的 91.8%
  • 5 月单用户月均 Token 使用量达 853 万，说明用户已将其用于较重的任务处理
  • 超 60% 活跃用户会切换基座模型或使用多模型对比功能

下载地址：

  • 国内版：https://www.tabbit-ai.com（接入国内顶级模型，免费使用）
  • 国际版：https://www.tabbitbrowser.com（接入国外主流模型）

二、常用功能（基础篇）

1. 多模型对话 —— 开箱即用

Tabbit 内置多款头部大模型（LongCat、DeepSeek、智谱 GLM、Kimi、Doubao、Qwen 等），全部免费使用。

  • 模型切换：根据任务场景灵活切换不同模型
  • 多模型对比：同个问题让多个模型同时回答，相互验证、获取灵感
  • 回答重试：对 AI 回答不满意时，可一键让其他模型重新生成，并保留历史版本便于对比

快捷键召唤 AI：Mac Cmd+] / Windows Ctrl+]

2. 网页阅读与总结

打开任意网页，点击右上角 Chat 按钮，AI 会自动以当前页面为上下文进行对话。支持：

  • 一键总结长文
  • 针对页面内容提问
  • 提取关键信息并结构化输出

3. Tab 自动整理

重度用户的”标签页焦虑”是 Tabbit 重点解决的问题：

  • 垂直标签栏设计（区别于传统顶部水平标签栏）
  • 智能分组：自动将打开的多个网页归类整理
  • 一键将整理结果加入书签，方便后续回访

4. 混合搜索模式

Tabbit 没有抛弃传统搜索。在使用 Chat 功能的同时，保留了传统搜索引擎入口，便于用户对比 AI 生成结果与传统搜索结果，兼顧效率与可靠性。

5. 智能截图

Tabbit 对截图功能做了深度优化：

  • 传统区域截图：支持拖拽选取
  • 智能吸附截图：自动识别页面 HTML 结构，自动吸附到页面模块边界，实现精准的点选截图，比传统截图更高效

三、高阶用法（进阶篇）

1. 多源引用 —— 打破信息孤岛

这是 Tabbit 最具特色的能力之一。在与 AI 对话时，上下文不仅限于当前页面，还支持引用：

引用来源 说明
当前页面 以当前浏览页面作为上下文
所有已打开的标签页 将全部打开的标签页作为上下文
指定标签页 通过 @ 符号引用特定标签页
收藏夹 引用收藏夹中的历史内容
本地文件 上传本地文件作为上下文

典型场景： 同时打开多篇不同来源的技术报道，通过 @ 引用这些页面并下达”生成对比表格”指令，AI 能准确提取各篇报道中的参数并进行结构化汇总，无需手动复制粘贴。

2. 妙招系统 —— 从指令到脚本的跃迁

「妙招」是 Tabbit 的自动化复用系统，支持三种形式：

① 提示词妙招

将高频 Prompt 封装存储，通过输入 / 快速召唤调用。例如：”微博文案生成”、”专业词汇翻译”。

② 脚本妙招（最强大）

用户用自然语言描述需求，AI 自动生成并运行 JavaScript 脚本。例如：

  • “提取页面所有图片” → 自动生成脚本并下载
  • “屏蔽这个网站的广告” → 生成去广告脚本
  • “修改网页配色” → 自定义网页显示效果

脚本妙招让不懂编程的用户也能通过自然语言”修改”网页，大幅降低生产力门槛。

③ 智能代理妙招

将 Agent 的操作流程（点击位置、数据填写、循环逻辑）记录沉淀为可复用脚本，后续直接调用。

创建方式： 执行完操作后点击页面上的小灯泡图标，输入妙招名称即可保存。调用方式： 输入 / 即可召唤。

3. Agent 自动化执行

这是 Tabbit 的核心竞争力。开启方式：首页对话框 → 设置 → 开启「智能代理」。

核心特点：

  • 自主拆解任务：自动将复杂任务拆解为多步执行计划
  • 跨网页执行：自动搜索信源 → 访问网页 → 提取关键点 → 输出带引用来源的报告
  • 标签组隔离：执行任务时自动将所需标签页整理到同一标签组，不影响用户其他操作
  • 视觉类操作：基于视觉识别操作网页，几乎不会被反爬机制拦截
  • Cookie 天然继承：AI 操作时直接以用户身份访问已登录网站（微博、淘宝、飞书等），无需额外配置权限

典型应用场景：

  • 从公众号后台采集文章数据（标题、阅读量、点赞等），自动填入飞书多维表格
  • 在小红书搜索房源信息，自动整理写入飞书文档
  • 批量在不同高校官网填写企业招聘资料
  • HR 场景：从筛选简历到生成 PPT，一句话完成

4. 记忆功能（1.0 新增）

Tabbit 1.0 新增了持久记忆能力：

  • 持续记录用户偏好、背景等重要信息
  • 形成”可调用记忆”，自动适配用户回复风格
  • 减少无效对话及重复动作

5. Skill 功能集成

Tabbit 支持三种 Skill 模式，可通过 Chat 辅助创建，实现更复杂的能力扩展：

  • 提示词（Prompt）模式：基础提示词封装
  • 脚本（Script）模式：执行自定义 JavaScript
  • Agent 模式：智能体自动化流程

6. 智能收藏夹

收藏夹具备语义搜索能力，收藏的内容可直接作为对话中的引用源，让历史积累的知识重新发挥作用。

Tabbit 的差异化优势：

  • 对国产模型生态的深度整合
  • 高度可定制的脚本能力（脚本妙招）
  • 中文语境下办公任务处理灵活性更高

五、收费模式

Tabbit 采用分级收费策略：

层级 内容 价格
永久免费 基础对话、网页阅读、常用妙招等核心功能 免费
订阅制（待推出） 高频 Agent 自动化调用、高级定制化场景 差异化订阅

Tabbit 负责人刘炯表示：面向大众用户的核心功能将永久免费；针对高算力的高级场景，将探索订阅制以平衡成本。

六、适用人群

推荐人群：

  • 🎯 重度信息工作者（编辑、分析师、研究员）：需要高频进行网页信息提取、对比和汇总
  • 💻 开发者与极客用户：对自定义脚本有需求，希望 AI 更深入介入浏览器交互
  • 📚 学生群体：用于文献阅读、多信源综述生成辅助
  • ♿ 无障碍需求用户：已有视障用户通过 Tabbit 直接生成专属 PPT 的实际案例

不太适合：

  • 仅有简单搜索需求、不习惯改变传统浏览器操作习惯的用户

七、总结

Tabbit 的核心价值在于：**它把浏览器从”网页承载工具”变成了”AI 工作入口”**。

通过重塑标签管理、收藏夹与跨页面对话逻辑，Tabbit 极大地缩短了用户从「获取信息」到「执行任务」的路径，实现了真正的工作流闭环。脚本妙招让非程序员也能”编程”，Agent 模式让复杂任务可以一句话搞定，多源引用让浏览器内的所有标签页变成一个流动的知识库。

作为美团在 AI 赛道的重要布局，Tabbit 目前仍处于快速迭代期。如果你是中重度浏览器用户，值得下载体验。
